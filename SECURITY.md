# Security Policy

## Supported status

RelayDesk is still in active early development. Security expectations should be interpreted accordingly, especially around provider integrations, terminal access, and local workspace execution.

## Reporting a vulnerability

Please do not open a public GitHub issue for security-sensitive problems.

When reporting, include:

- A short description of the issue
- Reproduction steps
- Impact assessment
- Any suggested mitigation

Until a formal security contact is added, treat the project as development-stage software and coordinate privately with maintainers before public disclosure.

## Scope notes

Please pay special attention to issues involving:

- Authentication and JWT handling
- Local terminal execution
- File system access
- Plugin execution boundaries
- Surrogate agent approval bypasses

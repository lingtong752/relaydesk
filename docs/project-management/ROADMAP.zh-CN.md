# RelayDesk 路线图

最后更新：2026-03-29

## 1. 当前阶段

当前项目处于 `Phase 6 起步 - 在补齐大部分 CLI 兼容层与工作台成熟度后，开始把插件系统和任务系统做成正式平台能力`。

当前阶段的目标不再只是“补兼容层”，而是把已经打通的基础能力收口成一版可提交、可对外说明的产品里程碑：

- 把任务工作台从只读摘要推进到正式模块
- 继续扩展插件运行时的权限边界和执行能力
- 补齐 Gemini Settings / MCP / 工具权限写回
- 增强替身 AI Agent 的运行轨迹、运行回放与 policy-engine
- 同步修正文档、测试报告和发布说明，确保仓库状态可直接提交

已完成：

- 新仓初始化与 monorepo 结构
- MongoDB 数据连接与基础集合
- JWT 鉴权
- 项目、会话、消息、运行 API
- WebSocket 实时订阅链路
- 登录页、项目页、工作台页
- 替身 AI Agent 启动/停止骨架
- 文件树与基础编辑器
- 浏览器终端工作台
- Git 状态、diff、暂存/提交、分支切换/创建工作台
- 替身 AI Agent 最小审批流
- 替身 AI Agent 人工接管与恢复运行
- 替身 AI Agent 运行检查点、审计事件与检查点级恢复
- WebSocket 自动重连与页面状态回补
- `provider-core` 统一 Provider 适配层
- Claude 真实 Provider 首条链路
- Codex 真实 Provider 首条链路
- Gemini 真实 Provider 首条链路
- 本机项目发现与 CLI 历史会话导入
- Claude / Codex / Gemini CLI 会话恢复与继续执行
- Claude / Codex Settings、MCP 与工具权限写回
- CodeMirror 多标签文件编辑与快速打开
- 多终端标签与重连回补
- Git fetch / pull / push 远程同步
- 插件安装、宿主上下文、本地 manifest 与受控动作执行
- 任务工作台与 TaskMaster 只读摘要
- 文档目录体系建立
- 开源治理与基础 CI/测试门禁

## 3. 当前优先级

P0：

- 任务工作台从只读摘要升级为可管理的执行面板
- 插件运行时权限分级、执行历史与后端 RPC
- Gemini Settings、MCP 与工具权限写回
- 提交前文档、测试报告和版本说明收尾

P1：

- 移动端观察体验
- 统一错误模型、加载骨架与空态
- 运行轨迹可视化增强
- RelayDesk 内建任务模型与 TaskMaster 双向同步预留

P2：

- 通知能力
- 任务与 PRD 深度管理
- 更丰富的插件市场与外部插件加载

P3：

- 团队协作与更长周期编排
- 更细粒度的 `policy-engine`
- 云端 / 自动化工作流扩展

## 4. 建议执行顺序

1. 第 1 周：同步文档、测试报告和发布说明，形成可提交版本基线
2. 第 2-3 周：把任务工作台从只读摘要升级为可管理的任务面板
3. 第 4 周：补 Gemini Settings、MCP 与工具权限写回
4. 第 5-6 周：扩展插件运行时权限分级、执行历史和本地后端能力
5. 第 7-8 周：增强替身运行轨迹、运行回放与 policy-engine
6. 第 9-10 周：移动端观察、通知与统一错误体验

## 5. 协作原则

- 新功能必须先补对应文档，再进入实现
- 核心状态模型改动时必须同步更新 PRD 和架构文档
- 每轮联调结束后补测试报告
- 里程碑完成后更新本路线图

# 文档中心

`docs` 目录用于集中管理 RelayDesk 的关键项目文档，避免产品、测试和项目推进信息散落在仓库各处。

## 目录结构

- `prd/`
  存放产品需求文档、范围定义、版本说明和功能边界。
- `testing/`
  存放测试报告、联调记录、冒烟验证结果和已知问题。
- `project-management/`
  存放路线图、迭代计划、里程碑、交付状态和开发优先级。
- `architecture/`
  存放架构设计、模块边界、数据模型和重要技术决策。

## 命名约定

- 长期维护的主文档使用稳定名称，例如 `PRODUCT_REQUIREMENTS.zh-CN.md`
- 带时间快照的报告使用日期后缀，例如 `TEST_REPORT-2026-03-28.md`
- 项目管理文档优先按用途命名，例如 `ROADMAP.zh-CN.md`、`BACKLOG.zh-CN.md`
